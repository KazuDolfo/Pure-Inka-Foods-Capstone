Manual Interactivo de Sistema

Pure Inka Foods - E-Commerce de Superalimentos

1. Instalación y Ejecución del Sistema

Sigue estos pasos para levantar el sistema localmente.

  • Node.js instalado.
  • Servidor MySQL activo.

Configuración de la Base de Datos:

  1. Dirígete a Backend/src/config.
  2. Abre el archivo bd.js (o .env) y actualiza el usuario, contraseña y nombre de la DB según tu entorno.
  3. Ejecuta en tu gestor MySQL:
    • DDL-3.sql
    • DML-v1-InkaFood.sql

Abre terminales independientes para Backend y Frontend y ejecuta:

npm install

Para el Backend:

npm run dev

Para el Frontend:

npm start

Pruebas automatizadas:

  • Backend: npm test
  • Frontend: ng test

Salud del API: Visita http://localhost:5000/api/health

3. Funcionalidades y Experiencia del Cliente

Al registrar una nueva cuenta, el sistema enviará un código de seis dígitos para validar la identidad de forma segura. Para pruebas, revisa la consola del backend para ver el código emitido.

La tienda muestra descripciones completas. Al ver un producto, el sistema sugiere automáticamente artículos similares relevantes.

Visualiza el número de productos agregados en tiempo real. Si agregas productos como invitado, estos se conservarán y sincronizarán automáticamente tras iniciar sesión.

  1. Dirección: Selecciona una guardada o agrega una nueva de forma dinámica.
  2. Tipo de Envío: Selecciona una de las 3 opciones (el costo se actualiza en el resumen).
  3. Métodos de Pago: Tarjetas (Stripe) y billeteras digitales (Yape, Plin).
  4. Comprobante: Elige emitir Boleta o Factura.

Historial completo de compras. Desde aquí puedes descargar tu comprobante de pago en formato PDF o cancelar un pedido si está en estado 'Procesando'.

4. Gestión del Administrador

Monitoreo dinámico de:

  • Ingresos reales.
  • Ventas promedio.
  • Cantidad de nuevos clientes.
  • Alertas visuales: Notificaciones si un producto alcanza stock crítico.

Control operativo total para gestionar, agregar o editar: stock, imágenes, precios y categorías de productos.

Actualiza los estados de envío (Procesando, Enviado, Entregado, Cancelado) para notificar al cliente del avance.

Activa o desactiva pasarelas y actualiza códigos QR o números de contacto para las billeteras digitales.

5. Auditoría y Seguridad

El sistema cuenta con un mecanismo que bloquea automáticamente la cuenta tras 5 intentos fallidos de inicio de sesión.

Todos los errores y registros de seguridad se pueden inspeccionar a detalle consultando los archivos generados en la carpeta logs del backend.
